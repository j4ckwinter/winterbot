## Spacing Scale

Declared values (multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline element padding |
| sm | 8px | Compact element spacing (avatar internals, badge padding) |
| md | 16px | Default element spacing (gap between dialogue portrait and text) |
| lg | 24px | Dialogue box internal padding |
| xl | 32px | Section padding increments |
| 2xl | 48px | Timeline entry vertical gaps |
| 3xl | 64px | Section padding-block (4rem = 64px, matches established pattern) |

Exceptions:
- Dialogue box bottom offset: 16px from viewport bottom edge (keeps it above mobile chrome)
- Dialogue box max-width: 640px centered — not part of 8-point scale but a layout constraint
- Avatar sprite grid: 16x16px base unit (pixel art grid), scaled up 2x to 32x32px rendered — aesthetic exception for pixel art fidelity
- JourneyNav gap between markers: 32px (2rem, matches current `gap: 2rem` in JourneyNav.astro)

Source: global.css body styles, Panel.astro padding (2rem = 32px), ExperienceSection.astro spacing patterns.

## Color

All color values are CSS custom properties. Hex values shown are Fire mode defaults (the `@theme` block). Leaf mode overrides are applied via `[data-theme="leaf"]` on `<html>`.

| Role | CSS Token | Fire Hex | Leaf Hex | Usage |
|------|-----------|----------|----------|-------|
| Dominant (60%) — world bg | `--color-world` | `#8B3A2A` | `#5A6B3A` | Page background, body bg |
| Secondary (30%) — surfaces | `--color-surface` | `#F9EFE5` | `#EFF5E8` | Panel bg, card bg, dialogue box bg, text on world |
| Border / text on surface | `--color-border` / `--color-text` | `#66203D` | inherited | Panel borders, body text on surface, timeline connectors |
| Accent primary (10%) | `--color-accent-primary` | `#A03020` | `#3D5A24` | Active nav marker, current quest marker, ACTIVE badge, focus outlines, dialogue box border |
| Accent secondary | `--color-accent-secondary` | `#D4854A` | `#8FA84A` | Inactive timeline markers, company/date text, achievement bullets, avatar neutral expression tones |
| Shadow | `--color-shadow` | `rgba(102,32,61,0.4)` | inherited | Panel drop-shadows, dialogue box shadow |

Accent primary reserved for:
- `.nav-marker.is-active` diamond and opacity state
- `.timeline-entry.is-current` entry-content border
- `.active-badge` background
- Focus-visible outline on all interactive elements (`outline: 3px solid var(--color-accent-primary)`)
- DialogueSystem box border when a new section fires (active state highlight)
- Blinking cursor `▌` in dialogue text (completed message state)

Destructive color: not applicable — Phase 3 has no destructive actions.

Source: global.css @theme block and [data-theme="leaf"] override; JourneyNav.astro focus-visible rule.

## Component Inventory

New components introduced in Phase 3:

### DialogueSystem (Preact island, `client:load`)

**Layout:**
- Fixed position, bottom of viewport: `position: fixed; bottom: 16px; left: 50%; transform: translateX(-50%)`
- Max-width: 640px; width: calc(100% - 32px) on mobile
- Z-index: 50 (below JourneyNav z-index: 100, above page content)
- Background: `var(--color-surface)`, border: 4px solid `var(--color-border)`, border-radius: 8px
- Box-shadow: 6px 6px 0px `var(--color-shadow)` (matches Panel aesthetic)
- Inner highlight ring: `::before` pseudo-element — 2px solid rgba(255,255,255,0.5) inset — mirrors Panel.astro

**Internal layout (RPG speech bubble):**
- Avatar portrait: left side, 32px wide, pixel art sprite (16x16 grid scaled 2x)
- Speaker name tag: above text body, `font-family: DotGothic16; font-size: 13px; color: var(--color-accent-primary)`; reads "GUIDE"
- Text body: `font-family: JetBrains Mono; font-size: 16px; line-height: 1.6; color: var(--color-text)`
- Blinking cursor `▌`: appended to text when typewriter is complete; blink via CSS animation `opacity 0.6s step-end infinite`
- `[A]` prompt indicator: bottom-right corner of text area, `font-family: DotGothic16; font-size: 11px; color: var(--color-accent-secondary); opacity: 0.7`

**States:**
- `typing` — characters rendering one by one, no cursor shown yet
- `complete` — full message visible, `▌` cursor blinking, `[A]` prompt visible
- `transitioning` — new section fired, previous message cleared, new message begins from char 0 immediately

**Interaction:**
- Click anywhere on dialogue box → skip to `complete` state instantly (DLG-05)
- `prefers-reduced-motion: reduce` → all messages appear in `complete` state immediately, no typewriter (DLG-06)

**Typewriter speed:** 40ms per character (deliberate readable pace, not jarring fast).

**Section messages (second-person, warm, game-flavored — DLG-05, DLG-06, DLG-07, DLG-08):**

### CSS Pixel Art Avatar

Built from `div` elements + `box-shadow` sprite technique (no external assets, no IP risk).

- Rendered size: 32×32px (16×16 pixel grid at 2x scale)
- Color palette: uses only design tokens — no hardcoded hex values
  - Skin/body: `var(--color-surface)`
  - Outline: `var(--color-border)`
  - Accent detail (hat, eyes): `var(--color-accent-primary)`
  - Secondary detail (scarf, cheeks): `var(--color-accent-secondary)`
- Expression states: controlled via CSS class on avatar wrapper
  - `.expr-neutral` — eyes: straight `—`, mouth: small line
  - `.expr-excited` — eyes: arcs `^`, mouth: open smile
  - `.expr-warm` — eyes: soft curves `~`, mouth: gentle smile

### Experience Scroll Reveal (EXP-04)

**Initial state (CSS):**
```css
.exp-entry {
  opacity: 0;
  transform: translateY(20px);
  transition: opacity 350ms ease-out, transform 350ms ease-out;
}
```

**Revealed state (class added by IntersectionObserver):**
```css
.exp-entry.is-visible {
  opacity: 1;
  transform: none;
}
```

**Reduced-motion override:**
```css
@media (prefers-reduced-motion: reduce) {
  .exp-entry {
    opacity: 1;
    transform: none;
    transition: none;
  }
}
```

IntersectionObserver threshold: 0.15 (entry is 15% visible before triggering).
Marker and content card animate as a single unit (no staggered reveal — D-15).

### JourneyNav Active State (scroll-driven)

The `<script>` block in JourneyNav.astro adds IntersectionObserver watching all 6 section IDs.

- On section enter: remove `is-active` from current marker, add `is-active` + `aria-current="location"` to the newly active marker; update diamond glyph from `◇` to `◆`
- Observer threshold: 0.5 (section must be 50% visible)
- Shares the same observer loop that powers DialogueSystem section tracking (single source of truth for active section)

## Interaction Contract

| Behavior | Trigger | Response | Duration | Motion |
|----------|---------|----------|----------|--------|
| Dialogue typewriter | Section enters viewport (IntersectionObserver) | Characters render one by one at 40ms/char | Variable (message length × 40ms) | Respects prefers-reduced-motion |
| Typewriter skip | Click on dialogue box | Jump to complete state instantly | 0ms | n/a |
| Dialogue section change | New section fires | Reset to char 0, begin new message | Immediate reset | Respects prefers-reduced-motion |
| Timeline entry reveal | Entry scrolls 15% into view | Fade in + slide up 20px | 350ms ease-out | Respects prefers-reduced-motion |
| Nav active update | Section 50% in view | Swap is-active class + diamond glyph | CSS transition 150ms | Always instant (nav state, not animation) |
| Avatar expression change | Section enters viewport | CSS class swap on avatar wrapper | Instant | n/a |
| Cursor blink | Dialogue complete state | CSS opacity step animation | 0.6s loop | Stops if prefers-reduced-motion |

## Accessibility Contract

| Concern | Implementation |
|---------|----------------|
| Dialogue box ARIA | `role="status"` with `aria-live="polite"` — screen readers announce new section messages without interrupting |
| Skip mechanism | `<button>` element wrapping dialogue text for keyboard accessibility (Enter/Space to skip) |
| Avatar | `aria-hidden="true"` — purely decorative |
| Speaker name | `aria-label="GUIDE says:"` on the name tag element |
| Reduced motion | All animations gate on `prefers-reduced-motion: reduce` — text appears immediately, entries appear pre-revealed |
| Focus management | Dialogue box does not trap focus; interactive only on click/keyboard activate |
| Nav ARIA | `aria-current="location"` updates dynamically on active section marker |
