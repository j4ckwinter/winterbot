FOUC script — exact implementation (is:inline, synchronous, first script in <head>):
  (function() {
    const saved = localStorage.getItem('theme');
    const theme = (saved === 'fire' || saved === 'leaf') ? saved : 'fire';
    document.documentElement.setAttribute('data-theme', theme);
    if (!saved) localStorage.setItem('theme', 'fire');
  })();

    Create `src/styles/global.css` with the complete Phase 1 token and base style system.

    The file structure must be:
    1. `@import "tailwindcss"` — must be first line
    2. `@theme { }` block — all tokens (generates both CSS variables AND Tailwind utility classes)
    3. `[data-theme="leaf"] { }` block — only the 4 values that differ; do NOT add a second @theme block
    4. Base styles — box-sizing reset, body background (world color + dot texture), font-family, heading scale

    CRITICAL rules:
    - Use `@theme {}` (not `:root {}`) for tokens — this is what generates `bg-world`, `text-text`, `border-border` utility classes in Tailwind v4
    - The `[data-theme="leaf"]` block uses plain CSS variable reassignment — NOT a second `@theme` block
    - Do NOT set `overflow: hidden` on body or any wrapper — this clips the Panel ::before highlight ring
    - The `--color-shadow` token uses `rgba()` syntax — keep it exactly as specified (Tailwind v4 handles rgba in @theme correctly)

    Use the exact token values from the interfaces block above. Do not round or adjust colors.
