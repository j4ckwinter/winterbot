  <action>
1. Add scroll-reveal CSS to `src/styles/global.css` (append after existing rules). These go in global.css (not scoped style) to prevent FOUC — Astro injects component scoped styles after JS, which would cause a brief flash of visible entries before JS hides them (per Pitfall 3 in RESEARCH.md):

```css
/* EXP-04: Scroll-reveal animation for experience timeline entries */
.exp-entry {
  opacity: 0;
  transform: translateY(20px);
  transition: opacity 350ms ease-out, transform 350ms ease-out;
}

.exp-entry.is-visible {
  opacity: 1;
  transform: none;
}

@media (prefers-reduced-motion: reduce) {
  .exp-entry {
    opacity: 1;
    transform: none;
    transition: none;
  }
}
```

Per D-13: fade + slide-up animation. Per D-16: 350ms ease-out. Per D-17: reduced-motion users see entries immediately.

2. Add a `<script>` block to `src/components/sections/ExperienceSection.astro` (after the closing `</style>` tag). Astro bundles `<script>` blocks as ESM modules that run after DOMContentLoaded:

```typescript
const expEntries = document.querySelectorAll('.exp-entry');
const revealObserver = new IntersectionObserver(
  (entries) => {
    entries.forEach((entry) => {
      if (entry.isIntersecting) {
        entry.target.classList.add('is-visible');
        revealObserver.unobserve(entry.target);
      }
    });
  },
  { threshold: 0.15 }
);
expEntries.forEach((el) => revealObserver.observe(el));
```

Per D-14: each entry animates individually when it scrolls into view. Per D-15: marker and content card animate together as a single unit (the `.exp-entry` `<li>` wraps both). `unobserve` after reveal prevents re-animation on scroll back (per Pitfall 4 in RESEARCH.md). Threshold 0.15 means entry must be 15% visible before triggering (per UI-SPEC.md).
