```typescript
const MESSAGES: Record<string, { text: string; expr: 'neutral' | 'excited' | 'warm' }> = {
  hero:       { text: "Welcome, Trainer! Your adventure begins here. Take a moment to get acquainted with your guide.", expr: 'excited' },
  about:      { text: "Ah, the Origin story. Every great journey starts somewhere -- this is where ours began.", expr: 'warm' },
```

IMPORTANT: The messages above use `--` for display in this plan. The actual messages in UI-SPEC.md use em-dashes. Read UI-SPEC.md for the canonical message text and use those exact strings.

Effects:
- `useEffect` on mount: listen for `sectionChange` CustomEvent on `window`, call `setSectionId(e.detail.id)`. Return cleanup that removes the listener.
- `useEffect` on `[sectionId]`: reset `charIndex` to `reducedMotion.current ? text.length : 0` (per D-03: immediate interrupt, per D-06/DLG-06: reduced-motion skips typewriter)
- `useEffect` on `[sectionId, isComplete]`: if not complete and not reducedMotion, run `setInterval(() => setCharIndex(i => i + 1), 40)`. Return cleanup `clearInterval`. 40ms per character per UI-SPEC.md.

Derived state:
- `const { text, expr } = MESSAGES[sectionId] ?? MESSAGES.hero;`
- `const isComplete = charIndex >= text.length;`
- `const displayed = text.slice(0, charIndex);`

Skip handler: `const skip = () => setCharIndex(text.length);` (per D-05/DLG-05)

JSX structure (per UI-SPEC.md layout):
```tsx
<div class="dialogue-system">
  <PixelAvatar expression={expr} />
  <button class="dialogue-box" onClick={skip} aria-label="Skip dialogue (click to reveal full message)">
    <span class="speaker-name" aria-label="GUIDE says:">GUIDE</span>
    <p role="status" aria-live="polite" class="dialogue-text">
```

**2. `PixelAvatar.tsx`** -- CSS pixel art avatar sub-component (per D-09, D-10, D-12)

Props: `{ expression: 'neutral' | 'excited' | 'warm' }`

Render a wrapper div with class `avatar-sprite expr-{expression}` and `aria-hidden="true"`, containing a child div with class `pixel-canvas`.

The avatar is a simplified 8x10 pixel face/head (per RESEARCH.md recommendation -- faces are expressive at 30-60 pixels). Use 3px per pixel unit, rendering at 24x30px. The `.pixel-canvas` element is `width: 3px; height: 3px;` and the parent `.avatar-sprite` has explicit `width: 24px; height: 30px; overflow: visible;`.

Design the pixel art character head using box-shadow on `.pixel-canvas`:
- All colors use CSS custom properties: `var(--color-border)` for outline, `var(--color-surface)` for skin/face, `var(--color-accent-primary)` for eyes/hat detail, `var(--color-accent-secondary)` for cheeks/scarf
- Neutral expression: straight horizontal line eyes, small line mouth
- Excited expression: arc eyes (^), open smile mouth
- Warm expression: soft curve eyes (~), gentle smile

Expression variants are applied via CSS class: `.expr-neutral .pixel-canvas`, `.expr-excited .pixel-canvas`, `.expr-warm .pixel-canvas` -- each redefines the `box-shadow` property with different pixel positions for eyes/mouth.

1. Add a `<script>` block to `src/components/nav/JourneyNav.astro` after the closing `</style>` tag. This Astro `<script>` block runs as an ESM module after DOMContentLoaded (Astro bundles it automatically):

```typescript
const sectionIds = ['hero', 'about', 'experience', 'projects', 'skills', 'contact'];
const sections = sectionIds.map(id => document.getElementById(id)).filter(Boolean) as HTMLElement[];

const navObserver = new IntersectionObserver((entries) => {
  const intersecting = entries.filter(e => e.isIntersecting);
  if (!intersecting.length) return;
  const most = intersecting.reduce((a, b) =>
    a.intersectionRatio > b.intersectionRatio ? a : b
  );
  const activeId = most.target.id;

  // Update nav markers -- swap is-active class, aria-current, and diamond glyph
  document.querySelectorAll('.nav-marker').forEach(el => {
    const isActive = (el as HTMLAnchorElement).getAttribute('href') === `#${activeId}`;
    el.classList.toggle('is-active', isActive);
    if (isActive) {
      el.setAttribute('aria-current', 'location');
    } else {
      el.removeAttribute('aria-current');
    }
    const diamond = el.querySelector('.diamond');
    if (diamond) diamond.textContent = isActive ? '◆' : '◇';
  });

  // Notify DialogueSystem Preact island via CustomEvent
  window.dispatchEvent(new CustomEvent('sectionChange', { detail: { id: activeId } }));
}, { threshold: [0, 0.25, 0.5, 0.75, 1.0] });

sections.forEach(s => navObserver.observe(s));
```

Multiple thresholds `[0, 0.25, 0.5, 0.75, 1.0]` handle sections taller than the viewport (per Pitfall 2 in RESEARCH.md). The highest `intersectionRatio` entry wins.
