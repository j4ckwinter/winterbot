## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding |
| sm | 8px | Compact element spacing, tab button gaps |
| md | 16px | Default element spacing |
| lg | 24px | Section padding, entry content padding |
| xl | 32px | Layout gaps |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level `padding-block: 4rem` per section |

Exceptions:
- Touch targets on tab buttons: minimum 44px tall at 320px viewport (per WCAG 2.5.5) — padding must expand to meet this at narrow widths
- JourneyNav: hidden via `display: none` at `max-width: 640px` — no spacing contract needed at mobile

Source: Existing `padding-block: 4rem` section rule in all section components; `gap: 0.5rem` tab list in `SkillsSection.astro`.

## Interaction Contract

Phase 4 introduces one new interaction: Skills tab filtering. All other interactions are carry-forwards from previous phases.

### Skills Tab Filter (new in Phase 4)

- Implementation: plain `<script>` block inside `SkillsSection.astro` (no Preact island)
- Trigger: click on any `.tab-btn[role="tab"]`
- Behavior:
  1. Set `aria-selected="true"` on clicked tab; set `aria-selected="false"` on all others
  2. Toggle `.tab-btn--active` class to clicked button, remove from others
  3. Update `aria-labelledby` on `#skills-panel` to match active tab ID
  4. Show `.skill-slot` elements where `data-category === activeCategory` OR activeCategory === `"all"`
  5. Hide non-matching slots via `display: none` (not `visibility: hidden` — must remove from tab order)
  6. Update diamond glyph text: active tab uses `◆`, inactive use `◇` — no, diamond glyphs are nav-only; tabs use text labels only
- Keyboard: tab key navigates between tab buttons; Enter/Space activates focused tab
- No animation on slot show/hide — instant visibility toggle
- prefers-reduced-motion: not applicable (no animation involved)

### Mobile Layout Fixes (PERF-02 scope)

These are not new interactions — they are responsive fixes to existing components.

| Component | Risk | Fix contract |
|-----------|------|-------------|
| `JourneyNav` | Fixed sidebar hidden at ≤640px | Already hidden via `display: none` — verify no bleed-through at 320px |
| `DialogueSystem` | Fixed bottom bar competes with content | Must not overlap scrollable content; verify `bottom: 0` stays within viewport at 320px |
| `SkillsSection` tab bar | 4 buttons may overflow at 320px | Tab list must wrap or compress; use `flex-wrap: wrap` if needed; no horizontal overflow |

### Carry-forward interactions (implemented, no changes)

- ThemeToggle: Fire/Leaf theme switch — no changes
- JourneyNav scroll observer: section tracking — no changes
- DialogueSystem typewriter + skip: section-contextual dialogue — no changes
- ExperienceSection scroll-reveal: `.exp-entry` IntersectionObserver — no changes
- ProjectCard hover lift: `translateY(-4px)` + shadow — no changes
- Skill slot tooltip: CSS-only `::before` on `:hover` / `:focus-visible` — no changes
